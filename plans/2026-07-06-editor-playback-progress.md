## Findings log

### 2026-07-06 — Part C/D complete + live smoke test (built app @47778, fresh server)

All verified in the real browser: new layout renders (sidebar w/ kbd chips, bottom track panel + bar navigator, slimmed toolbar); bar-navigator jump updates status bar; track switch (6str→4str) clamps
cursor; Del on lone-bar rest toasts "Bar is already empty"; R on rest toasts "Beat is already a rest"; fret entry dirties; **play-after-edit starts at cursor** (after alphaTab workaround below); dirty
exit shows 3-way modal; Save & leave saves (200) + navigates; edited tab plays in viewer with tempo 82 intact.

**Upstream alphaTab 1.8.0 bug (worth reporting):** `AlphaSynthWebWorkerApi.loadedMidiInfo` getter reads itself (`return this.loadedMidiInfo` instead of `this._loadedMidiInfo`,
dist/alphaTab.core.mjs:55187) → infinite recursion. `api.midiLoaded.on()` triggers it via the EventEmitterOfT fireOnRegister replay → RangeError, playback dead. Workaround in TabEditor.vue
`workAroundLoadedMidiInfoBug()`: probe `api.player.loadedMidiInfo` in try/catch, on throw redefine the prototype getter to return `_loadedMidiInfo`. Self-disables when upstream fixes it. Second trap:
midiLoaded REPLAYS the previous load synchronously at subscribe — one-shot handlers must ignore the replay (`armed` flag).

### 2026-07-06 — Part A diagnosis complete

**Playback verdicts:**

- Viewer playback of untouched tabs WORKS on fresh build + fresh server restart. The user-visible "playback regression" is explained by: backend caches index.html at startup (main.ts:70) + SPA
  catch-all returns 200 HTML for missing `/assets/*` (main.ts:752) → after any rebuild, a running server serves stale hashed URLs, the worklet chunk request gets HTML, `audioWorklet.addModule` fails
  with AbortError → all synth playback dead until server restart. Matches committed log `.playwright-mcp/console-2026-07-05T17-58-16-106Z.log`.
- **Tempo-loss bug (A4):** insert bar at index 0 in editor → save → reload: score.tempo 82→120 (default) + spurious 120 tempo automation on new bar 0. Song audibly starts at wrong tempo until bar 1
  (where the original 82 automation moved). All other playback data (playbackInfo, tunings, solo/mute, time sigs) survives the Gp7 round-trip intact.
- **Editor play-from-cursor bug (A5):** after any edit, Space plays from the stale previous position, not the cursor. `loadMidiForScore()` is async; the synchronous `tickPosition=` set right after it
  gets clobbered when the new MIDI loads.
